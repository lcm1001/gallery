## **Alternative: Use <img> tag instead of background-image**

If you want better control, replace the background-image with an actual `<img>` tag:

### **Old way (background-image):**

```html
<div class="product-image"
    style="background-image: url('images/white_flower.jpg');">
    <div class="product-badge">ORIGINAL</div>
</div>
```

### **New way (<img> tag):**

```html
<div class="product-image-container">
    <img src="images/white_flower.jpg"
        alt="White Flower Shine"
        class="product-image-actual">
    <div class="product-badge">ORIGINAL</div>
</div>
```

### **CSS for <img> method:**

```css
.product-image-container {
    width: 100%;
    height: 300px;
    position: relative;
    background-color: #fafafa;
    display: flex;
    align-items: center;
    justify-content: center;
    overflow: hidden;
}

.product-image-actual {
    max-width: 100%;
    max-height: 100%;
    width: auto;
    height: auto;
    object-fit: contain;
    display: block;
    margin: 0 auto;
}

.product-badge {
    position: absolute;
    top: 1rem;
    right: 1rem;
    background: rgba(26, 26, 26, 0.9);
    color: white;
    padding: 0.4rem 0.8rem;
    font-size: 0.75rem;
    letter-spacing: 1px;
    z-index: 10;
}

@media (max-width: 768px) {
    .product-image-container {
        height: 250px;
    }
}
```

## **Understanding the Difference:**

### **background-size: cover** (your current setting)

- Fills entire container
- May crop parts of image
- Good for landscape/consistent sized images

### **background-size: contain** (recommended fix)

- Shows entire image
- Centers it automatically
- Better for portraits and varied sizes
- ✅ **This is what you need!**

-----

## **Which Fix Should You Use?**

### **Quick Fix (5 minutes):**

Just change `background-size: cover` to `background-size: contain`

### **Better Fix (10 minutes):**

Use the complete CSS I provided above with `contain` and `background-repeat: no-repeat`

### **Best Fix (20 minutes):**

Switch to using `<img>` tags instead of background images - gives you more control
